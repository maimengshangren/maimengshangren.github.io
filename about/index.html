项目经历

RAG 智能问答平台

2026.01 — 2026.05

基于 Java 17 + Spring Boot 3.5 + Milvus / pgvector 的企业级 RAG 智能体系统，覆盖 文档入库 → 多路检索 → 意图识别 → 模型路由 → 流式输出 → MCP 工具调用 的完整链路。

技术栈：Java 17 · Spring Boot · RocketMQ · Redis · PostgreSQL · MyBatis-Plus

  • 多路检索引擎：通过多路检索引擎实现意图定向向量检索与全局向量检索的双路并行架构，并行执行各检索通道后通过去重、重排序等后处理流水线收敛结果，在保证召回率的同时提升检索精准度，解决单一检索方式覆盖率不足的问题。
  • 树形意图分类器：实现基于 LLM 的树形意图分类器，从缓存加载意图树，支持”知识库查询 / 工具调用 / 系统对话”三种意图类型，识别置信度不足时主动引导用户澄清，有效提升意图识别准确性。
  • 问题重写与拆分：基于 LLM 实现问题重写与拆分，支持多轮对话上下文补全和复杂问题拆解为多个子问题并行检索，配合术语归一化与规则兜底策略，解决用户口语化表达与知识库标准术语不匹配的问题，显著提升检索召回率。
  • 可编排文档入库：基于节点编排的文档入库流水线，支持数据获取、文档解析、文本分块、内容增强、向量索引等节点的可视化配置，每个节点独立执行并记录日志，支持条件跳转和拓扑环检测，实现灵活的文档处理流程。
  • 分布式公平排队限流：基于 Redis 有序集合 + 分布式信号量 + 发布订阅通知，实现跨实例的公平排队限流，通过 Lua 脚本保证队头抢占的原子性，支持 FIFO 排队与超时拒绝；配合 SSE 实时反馈排队结果与拒绝提示，有效控制高并发场景下的模型调用压力。
  • 三态熔断 + 模型降级链：实现三态熔断器对每个模型独立健康监控，配合优先级降级链与流式首包探测机制，当某模型连续失败达阈值时自动熔断并切换至下一候选，用户端流式响应无感知，保障多模型场景下的高可用性。
  • 多线程池隔离 + 跨线程上下文透传：按工作负载特征配置多个专用线程池，队列类型与拒绝策略差异化；通过跨线程上下文透传方案实现用户身份与链路追踪信息在异步场景下的完整传递，避免线程池间资源竞争。
  • AOP 全链路追踪：通过自定义注解 + AOP 切面实现 RAG 链路全链路追踪，维护调用层级栈并异步持久化至数据库，记录链路 ID、各节点耗时、输入输出、异常信息等，支持异步场景下的完整调用链可视化。
  • 会话记忆管理：实现滑动窗口 + 自动摘要的会话记忆机制，保留最近 N 轮对话，超过阈值时通过 LLM 异步生成摘要并持久化，在控制 Token 成本的同时保留关键上下文信息，避免长对话场景下的 Token 消耗过多问题。
